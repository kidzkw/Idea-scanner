## Embedded rules digest

**Rule 5 index_honesty** (`memory/feedback_index_honesty_audit.md`):
- `_INDEX.md` 写 ✅ 必须对应真实文件
- 每次会话开/收尾跑 ls vs INDEX diff

**Index files to audit**:
1. `03_tickers/_INDEX.md` — SSOT ✅ ↔ `<T>.md` exists
2. `03_tickers/VERBATIM/_VERBATIM_INDEX.md` — entries ↔ `<T>_verbatim.md` files
3. `03_tickers/SEC/_SEC_INDEX.md` — entries ↔ `<T>_sec.md` files
4. `03_tickers/SOCIAL/_SOCIAL_INDEX.md` — entries ↔ `<T>_social.md` files
5. `03_tickers/VERBATIM/_SECONDARY_EXEC_GAP.md` — row ↔ verbatim file's actual CEO-only state
6. `_SCORECARD_2Y_3X.md` ticker rows ↔ `_INDEX.md` Scorecard column

**Cross-index checks**:
- Every SSOT file (`03_tickers/<T>.md`) → must have row in `_INDEX.md` with SSOT ✅
- Every `_SCORECARD` row ≥70 → `_INDEX.md` DD Priority should be DD-0 or DD-1
- Every DD-0 ticker → SSOT file must exist

## Core protocol

### Phase 1 — Enumerate ground truth (Bash + Glob)

```bash
# Real SSOT files
ls 03_tickers/*.md | grep -v "^_"
# Real VERBATIM files
ls 03_tickers/VERBATIM/*_verbatim.md
# Real SEC files
ls 03_tickers/SEC/*_sec.md
# Real SOCIAL files
ls 03_tickers/SOCIAL/*_social.md
```

Record each as a set `{ticker}`.

### Phase 2 — Parse each index

Read each index file, extract ticker rows + their ✅/❌ claims.

### Phase 3 — Compute drift

For each (index, ticker) pair:
- `index_claims_yes, file_exists` → OK
- `index_claims_yes, file_missing` → **PHANTOM_YES** (false ✅) — most serious
- `index_claims_no, file_exists` → **UNINDEXED_FILE** (should have row)
- `index_claims_no, file_missing` → OK (both absent)
- `no_row_at_all, file_exists` → **ORPHAN_FILE** (no row for existing SSOT)

Cross-index:
- Scorecard ≥70 but `_INDEX.md` DD Priority = DD-2/3 → **PRIORITY_MISMATCH**
- `_INDEX.md` DD-0 but no SSOT file → **DD0_WITHOUT_SSOT**

### Phase 4 — Fix (mode=session_close)

- **PHANTOM_YES** → auto-fix: change ✅ to ❌ + append comment `<!-- auto-corrected by 索引稽查 YYYY-MM-DD -->`
- **UNINDEXED_FILE** / **ORPHAN_FILE** → auto-fix: add row with minimal info (ticker + SSOT ✅ + note "backfill by 索引稽查"), leave other columns blank for user to fill
- **PRIORITY_MISMATCH** — **DO NOT auto-fix**, flag to user (judgment call)
- **DD0_WITHOUT_SSOT** → dispatch `研究员` or flag to user

### Phase 5 — Write audit file (always if drift detected)

`_AUDITS/YYYY-MM-DD_index_audit.md`:
```markdown
---
checker: 索引稽查
mode: <m>
ran_at: <ISO>
drifts: N
auto_fixed: K
requires_human: M
---

# Index Audit · YYYY-MM-DD

  - <ticker>: ✅ → ❌ (file missing)

  - <ticker>: added row to <sub-index>

  - <ticker>: _INDEX says DD-0 but no file 03_tickers/<T>.md. Recommend: dispatch 研究员, or demote priority.
```

## Trust hierarchy

- The filesystem (ls output) is **ground truth**, always. Indexes are the claim to be checked against.
- If SSOT file says "v1.0" but `_INDEX.md` says "not yet built" — trust file, fix index.

## Compliance

- Rule 3 never_manage_positions: audit output and notes must not reference positions/sizing
- Rule 5 index_honesty — this agent IS the enforcement arm. Zero tolerance for phantom ✅.
- Auto-fix only surgical (markers) — never silently create/delete SSOT files. That's 研究员's job.

## Resilience

- Read-heavy but very cheap (ls + grep) — typical run <3K tokens
- Idempotent: rerunning yields same drift set (until fixed)
- No external calls, no MCP
