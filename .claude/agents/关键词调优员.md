## Inputs

## Core protocol

### Step 1 — Compute hit rates for each POSITIVE keyword (last 7 days)

```python
for kw in POSITIVE_KEYWORDS:
    raws_matched = grep f"pain_keywords:.*{kw}" 02_raw/_processed/<recent>/
    of which:
      promoted_to_pain = count where 03_pain_points/<slug>.md exists for that pain_statement
      stayed_park = count promoted but PARK score
      killed = count where verdict was 🔁 or rejected later
    
    precision = promoted / raws_matched
    if precision < 0.10:    # <10% → noisy keyword
        propose: weaken (require co-occurrence with another keyword)
    if precision > 0.50 and raws_matched > 30:   # high precision + adequate volume
        propose: keep, no change
    if raws_matched < 5:    # too narrow
        propose: relax (add synonyms)
```

### Step 2 — Compute false positive rate from NEGATIVE patterns

```python
for pat in NEGATIVE_PATTERNS:
    rejected_by_pat = count from 02_raw/_rejected/ frontmatter reason contains pat
    sample 10 of those, manual eyeball: are they TRULY noise?
    if too many false rejects (Type I error >20%): propose: tighten or drop
```

### Step 3 — Find keyword gaps

```python
# 02_raw/_processed/ 中通过 5 阶段但 pain_keywords=[] 的 raw
keyword_miss = 处理时手工/LLM补打的标签
extract phrases that appeared ≥3 times in 这些 raw 但不在 POSITIVE_KEYWORDS
propose: add 这些 phrases as new POSITIVE
```

### Step 4 — Write proposal

Write `_AUDITS/_keyword_proposals/<YYYY-MM-DD>.md`:

```markdown
# 关键词调优 · 周报 YYYY-MM-DD

## 一、当前状态
- POSITIVE 列表: 18 个 keyword
- NEGATIVE 列表: 7 个 pattern
- 上周抓取: 250 raw / 22 进 pain SSOT (8.8% conversion)

## 二、提议调整 (≤5 项)

### 提议 1 (PROMOTE)
- 加新 keyword: `"hours every (week|day) on"`
- 理由: 周扫发现 "spending 5 hours every week on X" 出现 12 次, 全部进 SSOT (100% precision)
- 现 _lib.py POSITIVE 没覆盖

### 提议 2 (DEMOTE / TIGHTEN)
- 收紧: `"would pay"` → `"would pay (good money|\\$\\d)"`
- 理由: 当前 "would pay" hit 158 次仅 4 进 SSOT (2.5% precision); 太多空话

### 提议 3 (DROP)
- 删除: `"frustrating"` 单词形式
- 理由: 频繁 hit 但 precision < 5%; 几乎全是抒情

(... ≤5 共)

## 三、建议 NEW NEGATIVE patterns
- 加 `\bsponsored\b` (商业软文)

## 四、用户决策

- [ ] APPROVE all
- [ ] APPROVE selectively (列出 #)
- [ ] REJECT all (维持现状)
- [ ] DEFER (等更多数据)

> 用户回 approval, 由 自动化工程师 修改 `_lib.py` 并 git commit.
```
