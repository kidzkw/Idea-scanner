## Embedded rules digest

**Extraction targets** (DD_WORKFLOW §1.4):
1. **AI Industry Fit 五条** (from SSOT §五 structure, v3.0):
   - AIF-1: AI ecosystem (positioning in AI value chain / customer mix)
   - AIF-2: Moat (pricing power / IP / switching cost)
   - AIF-3: AI finance (Rev YoY ≥30% / GM ↑3pp / FCF trajectory)
   - AIF-4: Production+beat (volume ramp / guidance beats)
   - AIF-5: R&D breakthrough (roadmap delivery / tape-out / qualification)
   Scan for data points relevant to each.
2. **Catalyst** — dated events (earnings / M&A / IPO / regulatory / product ship)
3. **Numeric facts** — revenue / margin / guidance / customer wins / unit shipments
4. **高管发言 (verbatim)** — any direct quote from CEO/CFO/COO/President/VP/GM
5. **§七 signals** — anything that triggers §七-A (preventive) or §七-D (time stop)

**Output**: a structured "extract sheet" appended to the raw file (not a new file), plus a trigger event for downstream.

## Inputs

- `raw_path`: absolute path to raw (accepted by Triage)
- `ticker`: from Triage receipt
- `source_weight`: from Triage

## Core protocol

### Step 1 — Full read

```python
body = Read(raw_path)  # full file now
```

### Step 2 — Extract AI Industry Fit-relevant data points

Grep for domain keywords (industry / segment / competitors / margin / cash).
For each hit, classify which SD it supports and whether it's supportive / neutral / contrary.

### Step 3 — Extract catalysts

Grep for date patterns (YYYY-MM-DD, "Q1 2026", "by end of year", "next quarter"). For each:
- Event type: earnings / product launch / M&A / IPO / regulatory / conference
- Binary-ness (✅ / ✅✅ / —)
- Source URL from frontmatter

### Step 4 — Extract numeric facts

Grep for `$X` / `Y%` / `Z million` patterns. For each:
- Metric name (revenue / margin / etc.)
- Value
- Period (Q1 2026 / FY26 / etc.)
- Context (guidance / actual / peer comparison)

### Step 5 — Extract verbatim quotes

Grep for quote delimiters ("..." / '...' / "say" / "commit" / "guide" / "expect"). For each found quote that attributes to a named exec:
- Speaker name + title
- Verbatim text
- Commit level (L1-L5 per §5.1)
- Context (earnings call / interview / conference)

If **any** verbatim found → set `verbatim_queued: true` in receipt → orchestrator dispatches 原话手 for retrofit/backfill check.

### Step 6 — §七 signal scan

Check for:
- §七-A preventive: thesis-break signals (SD-1 fails / key customer loss / material litigation / guidance cut)
- §七-D time stop: stale catalyst (committed milestone >X days overdue)
