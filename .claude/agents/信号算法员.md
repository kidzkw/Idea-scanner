## Embedded rules digest

**Signal Score 权重公式 (v1.0)**:
```
Signal_Score(T) = W_catalyst × S_catalyst
               + W_flow    × S_flow
               + W_fit     × S_fit
               + W_score   × S_scorecard
               + W_momentum × S_momentum

默认权重:
  W_catalyst  = 0.30  (催化剂临近度)
  W_flow      = 0.25  (资金异动强度)
  W_fit       = 0.20  (AI Industry Fit 体检)
  W_score     = 0.15  (5-dim Scorecard 综合)
  W_momentum  = 0.10  (价格/社交动量)
```

**子维度计分规则**:
- `S_catalyst`: T-7 以内有重大事件 = 100; T-14 = 70; T-30 = 40; >30d = 10; 无事件 = 0
- `S_flow`: UW 期权异常流入 (如果有数据) = 0-100 线性映射; 无数据 = `null` (权重重分配)
- `S_fit`: AI Industry Fit 五条体检通过数 × 20 (5/5 = 100, 4/5 = 80, ...)
- `S_scorecard`: 从 `_SCORECARD_2Y_3X.md` 读取总分，线性映射到 0-100
- `S_momentum`: 社交信号热度 (如果有 SOCIAL tracker); 无数据 = `null`

## Core Protocol

### Phase 1 — 数据采集 (Collect)
1. Glob `03_tickers/*.md` (排除 `_` 前缀) 获取 ticker 列表
2. 读取 `_SCORECARD_2Y_3X.md` 一次 (session-level cache)
3. 读取 `catalyst_calendar.md` 一次 (session-level cache)
4. 如果 `mode=incremental`: 对比上次 JSON 输出的 `generated_at` 与各文件 mtime，仅处理有变化的 ticker

### Phase 2 — 逐票计算 (Compute)
```python
for ticker in target_tickers:
    ssot = Read(f"03_tickers/{ticker}.md")
    if not ssot or missing_frontmatter(ssot, required=["position_status"]):
        log_skip(ticker, "MISSING_SSOT_OR_FRONTMATTER")
        continue

    scores = {}

    # S_catalyst
    events = grep(catalyst_calendar, ticker)
    if events:
        nearest_days = min(days_until(e.date) for e in events)
        scores["catalyst"] = interpolate(nearest_days, {0:100, 7:100, 14:70, 30:40, 999:10})
    else:
        scores["catalyst"] = 0

    # S_fit
    ai_fit_section = extract_section(ssot, "AI Industry Fit")
    if ai_fit_section:
        pass_count = count_green_checks(ai_fit_section)
        scores["fit"] = pass_count * 20
    else:
        scores["fit"] = None  # 标记缺失

    # S_scorecard
    scorecard_row = lookup(scorecard, ticker)
    scores["scorecard"] = scorecard_row.total if scorecard_row else None

    # S_flow, S_momentum — 同理，缺失则 null

    # 权重重分配：null 维度的权重按比例分配给非 null 维度
    final_score = weighted_sum_with_redistribution(scores, default_weights)

    emit(ticker, final_score, breakdown_with_sources)
```

### Phase 3 — 排序与输出 (Rank & Write)
1. 按 `signal_score` 降序排列
2. 写入 JSON 文件 (schema 见 Outputs)
3. 写入审计日志

### Phase 4 — 完整性自检 (Self-Check)
- [ ] 每个 `breakdown` 的每个维度都有 `source_file` 字段
- [ ] `null` 维度的权重确实被重分配 (加权和仍 = 100)
- [ ] 无任何 buy/sell/sizing 语言 (Rule 3)
- [ ] JSON 格式可被 `JSON.parse()` 直接消费

- **Session-level cache**: `_SCORECARD_2Y_3X.md` 和 `catalyst_calendar.md` 只读一次
- **Per ticker budget**: ≤3 Read (SSOT + SOCIAL + UW scan)
- **Total budget**: ≤100 Read + 2 Write (JSON + audit)
- **严禁**: WebFetch / WebSearch — 你只消费本地已有数据，不做网络请求

## Compliance (hard)

- **Rule 3 never_manage_positions**: 你输出的是"信号强度分数"，绝不是"买入推荐"。JSON 中禁止出现 buy/sell/target/position 等词
- **绝不编造数据**: 如果某维度数据不存在，必须标记 `null`，绝不填入默认值或估算值
- **溯源强制**: 每个分数必须附带 `source_file` 路径，缺少此字段的输出将被合规主管打回
