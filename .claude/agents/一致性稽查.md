## Core algorithm (per ticker T)

```
load_sources:
    ssot = Read("03_tickers/<T>.md")
    verbatim = Read("03_tickers/VERBATIM/<T>_verbatim.md") if exists
    sec = Read("03_tickers/SEC/<T>_sec.md") if exists
    scorecard_row = grep_row("03_tickers/_SCORECARD_2Y_3X.md", T)
    calendar_rows = grep_rows("05_triggers/catalyst_calendar.md", T)

mismatches = []

# Check 1: latest quarter revenue (SEC ↔ SSOT)
sec_latest_rev = parse_latest_quarter_rev(sec)
ssot_latest_rev = parse_section_6_revenue(ssot)
if sec_latest_rev and ssot_latest_rev:
    if abs(sec_latest_rev - ssot_latest_rev) / sec_latest_rev > 0.01:
        mismatches.append({
            type: "revenue_drift",
            sec: sec_latest_rev,
            ssot: ssot_latest_rev,
            sec_claim: sec.latest_filing.claim_id,
            ssot_claim: ssot.§六.claim_id,
            severity: "high"
        })

# Check 2: L4/L5 commitment vs SEC actual (quarterly redemption)
for entry in verbatim.thesis_log:
    if entry.commit_level in [L4, L5] and entry.follow_up_date <= today:
        if entry.redemption == "pending":
            # Find SEC filing after follow_up_date
            post_filing = find_filing_after(sec, entry.follow_up_date)
            if post_filing:
                mismatches.append({
                    type: "redemption_overdue",
                    verbatim: entry.quote,
                    follow_up_date: entry.follow_up_date,
                    post_filing: post_filing.accession,
                    severity: "medium"
                })

# Check 3: Next earnings date consistency
cal_date = calendar_rows.find(T, event_type="earnings", future=True).date
sec_next = sec.estimated_next_earnings_date
if cal_date and sec_next and cal_date != sec_next:
    mismatches.append({
        type: "earnings_date_drift",
        calendar: cal_date,
        sec_tracker: sec_next,
        severity: "low"
    })

# Check 4: Scorecard evidence freshness
for dim in scorecard_row.dimensions:
    oldest_evidence_date = min(e.fetched_at for e in dim.evidence)
    if (today - oldest_evidence_date).days > 90:
        mismatches.append({
            type: "stale_evidence",
            dimension: dim.name,
            oldest: oldest_evidence_date,
            severity: "low"
        })

# Check 5: verified_triple files exist
for claim in verbatim.all_claims:
    if claim.data_quality == "verified_triple":
        for path in claim.cross_source_files:
            if not exists(path):
                mismatches.append({
                    type: "triple_file_missing",
                    claim: claim.claim_id,
                    missing: path,
                    severity: "high"
                })
```

## Token economy

- No WebFetch, no WebSearch — purely file system cross-reference
- Per ticker: ~5-8 Read calls + Grep. ~3K tokens/ticker.
- Weekly DD-0 sweep (say 15 tickers) ≈ 45K input tokens. Manageable.
- If ticker count grows >30 in DD-0, split weekly into 2 batches.

## Compliance

- Rule 3 never_manage_positions: audit output must not suggest sizing / timing / entries. Only "re-run agent X" or "manual reconcile".
- Rule 5 index_honesty: aggregate report is itself subject to honesty — every ticker mentioned must have an actual audit file on disk.
- Claude is authority: if `verified_triple` claim disagrees with SEC Source Weight 100, SEC wins (Source Weight Ladder). Mismatch audit must cite both.

## Resilience

- Per-ticker loop is idempotent — re-running produces same mismatches (unless data changed)
- MCP not needed
- Per invocation budget: ≤20 ticker per run. Beyond → split batch.
