## Embedded rules digest

**三类核心实体 (Entity Model v1.0)**:

```
┌─────────────┐     N:N      ┌──────────────┐
│   Ticker    │─────────────│   Sector     │
│  (SSOT.md)  │             │  (主题/赛道)  │
└──────┬──────┘             └──────────────┘
       │ 1:N
       ▼
┌─────────────┐     N:1      ┌──────────────┐
│   Quote     │─────────────│   Event      │
│ (VERBATIM)  │             │  (催化剂)    │
└─────────────┘             └──────────────┘
```

**关系类型 (Edge Types)**:
- `supplies_to(A, B)`: A 是 B 的上游供应商 (例: COHR supplies_to NVDA)
- `competes_with(A, B)`: A 和 B 是同赛道竞争者
- `sector_member(T, S)`: T 属于 S 赛道 (多对多)
- `quote_anchors(Q, T)`: Quote Q 锚定到 Ticker T 的 SSOT
- `event_triggers(E, T)`: Event E 是 Ticker T 的催化剂

## Core Protocol

### Phase 1 — 实体采集 (Entity Collection)
1. Glob `03_tickers/*.md` 获取全部 ticker 节点
2. 从每个 SSOT 的 frontmatter 提取: `sector`, `value_chain_layer`, `customers` (如有), `suppliers` (如有)
3. 读取 `01_value_chain/` 下的赛道定义文件，构建 Sector 节点列表
4. 缓存所有 frontmatter 数据 (session-level, 避免重复读取)

### Phase 2 — 关系推断 (Edge Inference)
```python
for ticker in tickers:
    ssot = read_frontmatter(f"03_tickers/{ticker}.md")

    # 直接关系 (from SSOT explicit fields)
    if ssot.customers:
        for customer in ssot.customers:
            emit_edge(ticker, customer, "supplies_to", confidence="ssot_explicit", source_file=ssot.path)

    if ssot.suppliers:
        for supplier in ssot.suppliers:
            emit_edge(supplier, ticker, "supplies_to", confidence="ssot_explicit", source_file=ssot.path)

    # Sector 映射
    for sector in ssot.sectors:
        emit_edge(ticker, sector, "sector_member", source_file=ssot.path)

    # 竞争关系 (同 sector + 同 value_chain_layer 的 tickers)
    peers = find_peers(ticker, same_sector=True, same_layer=True)
    for peer in peers:
        emit_edge(ticker, peer, "competes_with", source_file=ssot.path)
```

**关键决策点**:
- 如果两个 ticker 只是同 sector 但不同 layer → `supplies_to` 而非 `competes_with`
- 如果关系来自 VERBATIM 引用而非 SSOT 显式字段 → 标记 `confidence: verbatim_inferred`
- 如果关系来自你的外部知识而非任何文件 → **必须**放入 `unverified_edges`，标记 `reason: inferred_from_knowledge_not_file`

### Phase 3 — Quote 锚定 (Quote Anchoring)
```python
for ticker in tickers:
    verbatim = Read(f"03_tickers/VERBATIM/{ticker}_verbatim.md")
    if not verbatim:
        continue

    for quote_block in extract_quote_blocks(verbatim):
        emit_edge(
            from=quote_block.quote_id,
            to=ticker,
            type="quote_anchors",
            hash=quote_block.verified_quad_hash,  # 必须使用原始 hash
            raw_text_excerpt=quote_block.text[:100],  # 前 100 字，不改写
            source_file=f"03_tickers/VERBATIM/{ticker}_verbatim.md"
        )
```

### Phase 4 — 输出与自检 (Output & Validate)
- [ ] 每条 edge 都有 `source_file` 字段
- [ ] `unverified_edges` 数量 < 总 edge 数的 20% (否则告警)
- [ ] Quote 锚定使用了原始 hash，未做任何改写
- [ ] JSON 格式可被 `JSON.parse()` 直接消费

## Token Economy

- **Session-level cache**: 所有 SSOT frontmatter 在 Phase 1 统一读取，后续不重复
- **VERBATIM 按需读取**: 仅在 `link_quotes` 任务时读取，其他任务跳过
- **Per ticker budget**: ≤2 Read (SSOT + VERBATIM)
- **Total budget**: ≤80 Read + 2 Write (JSON + audit)
- **严禁**: WebFetch / WebSearch — 图谱仅基于本地已有的 SSOT 数据构建

## Compliance (hard)

- **Rule 3 never_manage_positions**: 图谱是纯结构化关系描述，绝不包含任何 buy/sell 语言
- **绝不改写原话**: Quote Engine 的核心原则。如果你发现自己在"概括"一段 verbatim → 立即停止，使用原文
- **溯源强制**: 每条 edge 必须附带 `source_file`。缺少此字段 → 自动归入 `unverified_edges`
- **不猜测关系**: 如果 SSOT 没有明确写出 A 供应给 B → 不要根据你的知识推断。标记 `unverified`
