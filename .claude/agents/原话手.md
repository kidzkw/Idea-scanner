## Core protocol: Three-Way Parallel

### Step A — Probe MCP (R1b, 4-way)

Call `mcp__chrome-devtools__list_pages`. Find 4 tabs: chatgpt.com / grok.com / **gemini.google.com** / notebooklm.google.com. For each: alive or dead. Build status matrix:

- **4 alive** → full **verified_quad** 标准
- **3 alive** (任意 3) → verified_triple
- **2 alive** (任意 2) → verified_dual
- **0-1 alive** → primary-only mode, 缺的那几路入 `_VERBATIM_BACKFILL_TRACKER.md`

### Step B — Watchdog (R1a)

Write lock `03_tickers/VERBATIM/_lock_<TICKER>_<event>.txt` with:
```
step=5 ticker=<T> event=<E> mcp_state=alive|down stage=probe|claude|chatgpt|grok|merge started=<ISO>
```
Update `stage=` as you progress. Remove on completion.

### Step C — Claude path (always, primary)

WebSearch + WebFetch queries:
- `"<TICKER> <CEO name> earnings call transcript <quarter>"`
- `"<TICKER> CEO interview site:cnbc.com OR site:bloomberg.com"`
- `"<TICKER> investor presentation transcript <event>"`

WebFetch top 3 transcript URLs (Seeking Alpha / Motley Fool / Investing.com / IR press release) — these are Source Weight 85-95.

Extract 5-15 quotes minimum. **Secondary exec MANDATORY** — capture CFO / COO / President / VP / GM. CEO-only = add row to `03_tickers/VERBATIM/_SECONDARY_EXEC_GAP.md` with priority flag.

For each quote capture: speaker name + title, date, event, verbatim quote (no paraphrase), commit level (L1-L5), context, source URL.

Write raw: `02_raw/_processed/YYYY-MM/YYYY-MM-DD_<TICKER>_claude_verbatim-<event>.md` (per FILENAME_FORMAT_v1.md §1; pre-earnings 用 `pre-earnings-verbatim` token)

### Step D — Parallel ChatGPT + Grok + **Gemini** (if MCP alive, v3.1 四方)

Dispatch in parallel (**three tool calls in one message**, 同时 fire):

**ChatGPT tab** via chrome-devtools — DD_WORKFLOW §5.4 ChatGPT 模板 (结构化 L1-L5 extractor).

**Grok tab** via chrome-devtools — Expert-mode (30-90d signals + X posts + 删推归档).

**Gemini tab (gemini.google.com) ⭐ v3.1 新加** — 利用 Google Search grounding:
```
Using Google Search, find <TICKER> <CEO/CFO name> earnings call transcript for Q<N> <YYYY>.
Extract verbatim executive quotes by commit level (L1-L5 from 'excited about' to 'will deliver $X').
Each quote: verbatim text + speaker name+title + event date + context + clickable source URL.
Minimum 5 L3+ commitments. If no public record, say so — do NOT fabricate.
Also flag any tone shift vs prior quarter (e.g. 'far exceed' → 'in line with').
```

Save each output as separate raw:
- `02_raw/_processed/YYYY-MM/YYYY-MM-DD_<TICKER>_chatgpt_verbatim-<event>.md`
- `02_raw/_processed/YYYY-MM/YYYY-MM-DD_<TICKER>_grok_verbatim-<event>.md`
- `02_raw/_processed/YYYY-MM/YYYY-MM-DD_<TICKER>_gemini_verbatim-<event>.md` ⭐ 新

Gemini 权重 60 (regular chat) 或 70 (若 toggle Deep Research mode). 检查所有 output 的 truncation.

### Step E — Merge & arbitrate (Opus judgment, 4-way)

Cross-reference quotes across **4 sources** (Claude + ChatGPT + Grok + Gemini). For each quote:
- `verified_quad`: 4 sources independently produce same quote ⭐ 最高 tier
- `verified_triple`: 3 sources
- `verified_dual`: 2 sources
- `primary_only_pending`: only Claude (MCP down or others didn't have it)
- `single_non_primary`: 只 ChatGPT/Grok/Gemini 任一 有, Claude 没 → **NOT anchor**, flag for review
- `contradicted`: 源之间不一致 → Source Weight Ladder (SEC 100 > IR 95 > Transcript 90 > NotebookLM 85 > ... > Gemini 60 > Grok 50)

### Step F — Write to `<TICKER>_verbatim.md`

Append Thesis Log entries per `DD_AUTHORITY_v3.2.md §5.3` format. Each entry: speaker, title, event, verbatim quote + commit level tag, context, source URL, follow-up date, `data_quality` label.

### Step G — Propagate to SSOT

For L4/L5 commitments, update `03_tickers/<TICKER>.md`:
- §四 Catalyst — add follow-up date + expected-vs-actual row
- §七-A — preventive stop-loss trigger if commit missed

### Step H — Compliance audit

Run mini-checklist:
