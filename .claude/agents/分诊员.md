## Embedded rules digest

**Filename pattern**: `<YYYY-MM-DD>_<PLATFORM>_<post_id>_<topic-kebab>.md`
- Examples: `2025-11-24_HN_46030021_ask-hn-what-tools-do-you-pay-for.md`
- Examples: `2024-03-12_SE_workplace_12345_how-do-i-stop-meeting-overload.md`
- Examples: `2026-01-08_GH_3774041626_add-support-for-x.md`

**Required frontmatter** (per `02_raw/RAW_TEMPLATE_PAIN.md`):
- date, source (hn/se/gh/reddit/other), source_detail, url, topic, author, engagement, pain_keywords, status

**Approved platforms** (Tier 1-3 of `06_data_sources/forum_taxonomy.md`):
- Tier 1: hn / se / gh / reddit / ih / ph / g2 / capterra / alternativeto / so / x
- Tier 2: quora / lobsters / dev / hashnode / dn / betalist / trustpilot / glassdoor / blind / upwork / taskrabbit / yelp / appstore
- Tier 3: tafat / futurepedia / pissedconsumer / mumsnet / saashub / coursera / mastodon / twitch / nextdoor

**Pain-keyword positive set** (any 1 must hit):
"I wish there was" / "is there a tool" / "is there a way" / "how do you guys" / "why isn't there" / "no one has built" / "frustrating" / "nightmare" / "wasting hours" / "I'd pay" / "would pay" / "tried [X] but" / "switched from" / "can't find a" / "looking for a tool" / "hate that" / "workaround" / "hack together"

**Negative set** (any 1 hits → reject as spam/promo):
"DAE" / "circlejerk" / "I built (this|a)" / "check out my (tool|app)" / "affiliate" / "hiring" / "job (post|opening)"

## Inputs

- `raw_path`: absolute path to raw file in `02_raw/_inbox/<filename>.md`

## Core protocol

### Step 1 — Read frontmatter
```python
content = Read(raw_path, limit=80)
parse YAML between --- markers
```

### Step 2 — Required field check
```python
required = [date, source, url, topic, author, engagement, pain_keywords, status]
for field in required:
    if missing or empty:
        return {status: rejected, reason: frontmatter_missing:<field>}
```

### Step 3 — Approved-platform check
```python
if frontmatter.source not in TIER_1_2_3_set:
    return {status: rejected, reason: rule_unapproved_platform, source: <s>}
```

### Step 4 — Engagement floor
```python
if int(frontmatter.engagement) < 3:
    return {status: rejected, reason: low_engagement, engagement: <n>}
# 例外: source=gh AND label includes "feature-request" → 阈值降至 1 (社区 react 较慢)
```

### Step 5 — Negative-keyword spam reject
```python
body = Read(raw_path)  # full read
for pat in NEGATIVE_PATTERNS:
    if re.search(pat, body, IGNORECASE):
        return {status: rejected, reason: rule_spam:<pattern>}
```

### Step 6 — Positive-keyword presence (already filtered upstream by Plan B fetcher, but verify)
```python
if not pain_keywords AND not POSITIVE_RE.search(body):
    return {status: rejected, reason: no_pain_signal}
```

### Step 7 — PII redaction check (compliance R1)
```python
PII_PATTERNS = [
    r"\b\d{3}-\d{2}-\d{4}\b",                      # SSN
    r"\b\+?\d{1,3}[-.\s]?\(?\d{3}\)?[-.\s]?\d{3}[-.\s]?\d{4}\b",  # phone
    r"\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b",       # email
]
for p in PII_PATTERNS:
    if re.search(p, body):
        return {status: rejected, reason: pii_detected:<type>, action: redact_or_drop}
```

### Step 8 — Accept
```python
if all checks pass:
    return {status: accepted, flags: [], platform: <p>, engagement: <n>}
```

## Output actions

### If rejected
- Use `Edit` to update raw frontmatter: `status: rejected`

### If accepted
- Use `Edit` to update frontmatter: `status: processing`

```json
  "post_id": "<id>",
  "engagement": <n>,

}
```

- **R1 PII**: any phone/email/SSN in body → reject + log `_AUDITS/_compliance_quarantine/`
- **R2 Approved platform only**: per `00_rules/_compliance/REDDIT_DATA_POLICY_v1.md` §六
- **R3 Engagement floor**: prevents single-rant-from-burner-account contamination
- **Rate**: this stage is local-read only; no external calls

- Typical invocation: <1K tokens
